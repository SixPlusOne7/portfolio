University of Hawai'i Drone Team (UHDT)

LiDAR Mount

  • Designed a modular mount system to secure the LiDAR unit, power bank, Jetson, and battery on the drone.
  • Used dovetail joints to connect parts too large to fit on the 3D printer's build plate.
  • Modular structure improves assembly and maintenance while keeping the design strong and lightweight.

Landing Gear

  • Redesigned the landing gear to allow the UAV to fit into standard luggage for travel.
  • Replaced fixed carbon fiber legs and PLA parts used in previous years, which frequently fractured during landings.
  • Selected TPU for its excellent shock absorption, durability, and flexibility under dynamic loads.
  • During testing, TPU was too flexible, causing the drone to wobble upon landing.
  • Plan to prototype with PETG next, which offers better stiffness and impact resistance for more stable landings.

Coding Projects

Kinematic Analysis of Undulatory Swimmers for Bio-inspired Robotics

Python tool to quantify swimming mechanics and guide energy-efficient robot design

I wrote this code to analyze video frames of undulatory swimmers, such as fish and bio-inspired robots, to study their body shapes and movement patterns. By tracking features such as head angle and swimming speed, the code can be used to better understand underwater locomotion and support the development of robotic models that mimic efficient biological movement.

  • Body Shape Extraction: Track body midlines by marking points from head to tail.
  • Curve Generation: Convert points into smooth curves using interpolation.
  • Movement Analysis: Measure head angle and speed to study movement efficiency.
  • Data Visualization: Plot curves and data points to compare body shapes.

Tools: Python, OpenCV, SciPy, Matplotlib

Bank Record Management System (C Project)

I developed a terminal-based C program that manages a list of bank account records. It allows users to add, find, delete, and display records, and saves data to a file for persistent storage. I used a linked list to store the data in memory, and implemented input validation to make sure the program handles all user input safely without crashing or misbehaving.

  • Language: C
  • Concepts: Pointers, Structs, Linked Lists, File I/O, Memory Management
  • Input Handling: Thorough input validation ensures safe handling of account numbers, names, and addresses. For example, invalid input is rejected and the user is re-prompted until valid input is received.
  • Structure:
    • user_interface.c: Handles the user menu, input logic, and function calls
    • database.c: Implements core functions like addRecord, findRecord, deleteRecord, printAllRecords, readfile, and writefile
    • record.h: Defines the structure for each record
    • database.h: Contains function prototypes shared across files
  • Reliability: All core features were tested successfully. The program reads and writes data in the correct format and maintains memory integrity by cleaning up all dynamically allocated memory on exit.
